MySQL高级

账户管理
  • 服务实例级账号：，启动了一个mysqld，即为一个数据库实例；如果某用户如root,拥有服务实例级分配的权限，那么该账号就可以删除所有的数据库、连同这些库中的表
  • 数据库级别账号：对特定数据库执行增删改查的所有操作
  • 数据表级别账号：对特定表执行增删改查等所有操作
  • 字段级别的权限：对某些表的特定字段进行操作
  • 存储程序级别的账号：对存储程序进行增删改查的操作
权限管理
  • 实例级账号上执行，即root。
  • 所有用户及权限信息存储在mysql数据库的user表中，查看user表的结构:desc user\G;
  • 查看所有用户:select host,user,authentication_string(密码，加密后的值) from user;

  • 创建账户并授权：grant 权限列表 on 数据库 to ‘用户名’@’访问主机’ identified by ‘密码’;

  • 授权：grant 权限名称 on 数据库 to 账户1,账户2,… with grant option;

  • 查看用户有哪些权限:show grants for py;
  • 回收权限

-- 求所有电脑产品的平均价格,并且保留两位小数
    select round(avg(price),2) from goods;
-- 查询所有价格大于平均价格的商品，并且按价格降序排序
    select * from goods where price > (select round(avg(price),2) from goods) order by price desc;
-- 查询类型为'超极本'的商品价格
    select * from goods where cate='超级本';
-- 查询价格大于或等于"超级本"价格的商品，并且按价格降序排列
    select * from goods where price >=any (select price from goods where cate='超级本')

-- 基本查询

  -- 比较运算符

      等于，默认是等于判断，没有运算符
      小于$lt  less than
      小于或等于$lte less than equals
      大于$gt great than
      大于或等于$gte great than equals
      不等于$ne not equals
      例1：查询名称等于'郭靖'的学生

      db.stu.find({name:'郭靖'})

      -- 查询年龄小于30岁
      db.stu.find({age:{$lt:30}})
      db.stu.find({age:{$lte:20}})

      -- 查询不等于18岁
      db.stu.find({age:{$ne:20}})

  -- 逻辑运算符

      查询时可以有多个条件，多个条件之间需要通过逻辑运算符连接
      逻辑与：默认是逻辑与的关系

      -- 查询18的男生
      db.stu.find({age:18,gender:true})

      -- 查询18的学生或者家乡为大理的学生
      db.stu.find({$or:[{age:18},{hometown:'大理'}]})


      例3：查询年龄大于或等于18，并且性别为true的学生
      db.stu.find({
          $or:[{age:{$gt:18}},{age:18}],
          gender:true
      })


  -- 范围运算符

      使用"$in"，"$nin" 判断是否在某个范围内
      例6：查询年龄为18、28的学生

      db.stu.find({age:{$in:[18,20]}})

      db.stu.find({age:{$nin:[18,20]}})


  -- 支持正则表达式

      mongodb提供的终端shell，同时也是一个js的执行器，可以编写js的代码和函数
      使用//或$regex编写正则表达式
      例7：查询姓黄的学生
      db.stu.find({name:/黄/})
      db.stu.find({name:{$regex:'^黄'}})

      如：一个文档有5个字段，需要显示只有3个，投影其中3个字段即可
      语法：
      参数为字段与值，值为1表示显示，值为0不显示

  -- 聚合 aggregate

      聚合(aggregate)主要用于计算数据，类似sql中的sum()、avg()
      语法
      db.集合名称.aggregate({管道:{表达式}})

      $group

      将集合中的文档分组，可用于统计结果
      \_id表示分组的依据，使用某个字段的格式为'$字段'
      例1：统计男生、女生的总人数

      db.stu.aggregate(
          {
              $group:
              {
                  \_id:'$gender',
                  count:{$sum:1}
              }
          }
      )
      按照性别分组，求每组的年龄总数
      db.stu.aggregate( { $group: { \_id:'$gender', count:{$sum:'$age'} } } )

      按照性别分组，求每组的平均年龄
      db.stu.aggregate( { $group: { \_id:'$gender', count:{$avg:'$age'} } } )

      按照性别分组，把年龄组成数组
      db.stu.aggregate( { $group: { \_id:'$gender', count:{$push:'$name'} } } )

      按照性别分组，把整个文档组成数组
      db.stu.aggregate( { $group: { \_id:'$gender', count:{$push:'$$ROOT'} } } )

                  counter:{$sum:1},
                  avgAge:{$avg:'$age'}

      db.stu.aggregate(

      )

      修改输入文档的结构，如重命名、增加、删除字段、创建计算结果
      例1：查询学生的姓名、年龄

  -- 分组后再投影

      )

  -- $sort

      将输入文档排序后输出
      例1：查询学生信息，按年龄升序
      db.stu.aggregate({$sort:{age:1}})


  -- $skip

      跳过指定数量的文档，并返回余下的文档
      例2：查询从第3条开始的学生信息
      db.stu.aggregate({$skip:2})

          {$sort:{counter:1}},
          {$skip:1},
          {$limit:1}
      )
      注意，必须先skip，再limit

  -- $unwind

      将文档中的某一个数组类型字段拆分成多条，每条包含数组中的一个值

      对某字段值进行拆分

数据结构

  • 数据结构：是计算机存储、组织数据的方式。数据结构是指相互之间存在一种或多种特定关系的数据元素的集合。数据结构会决定算法。

顺序表

  • 数据地址：申请一大块连续的内存，依次存入数据，从而维护数据之间的顺序关系。

  • 表头地址：在数据地址上再多个表头地址。

  • 一体式结构：表头与数据申请连续地址。

  • 分离式结构：不连续，表头地址最后会存储数据地址。常用这种方式。

  • 数据地址(元素外置)：通过存储数据地址来查找数据，存储不同的数据类型。

  • 顺序表

链表

  • 链表：在每一个节点（数据存储单元）里存放下一个节点的位置信息（即地址），即=数据+下一节点的地址
  • 单向链表：包含两个域，一个(A)域存储数据，一个(B)域存储下一个节点，最后一个节点的(B)域不存储信息。
  • 双向链表：包含三个域，一个(A)域存储上一个节点，中间(B)域存储数据，最后一个(C)域存储下一个节点，最后一个节点的(C)域不存储信息。
  • 单向循环链表：在单链表的基础上，最后一个节点的(B)域存储头节点。
  • 栈：只允许在一端进行操作，按照后进先出原理运作。
  • 队列：只允许在一端进行插入操作，而在另一端进行删除操作，按照先进先出的原理运作。
  • 双端队列：元素可以从两端弹出，其限定插入和删除操作在表的两端进行。双端队列可以在队列任意一端入队和出队。

排序算法

  • 稳定性：定排序算法会让原本有相等键值的纪录维持相对次序。

  • 冒泡排序：重复遍历，从左第一个数开始，把大的数往后排。

  • 选择排序：从左拿出最小的数往后比，遍历完把最小数放最左。

  • 插入排序：从右边选择小的数往左边比较，最小的往最左挪组成有序队列(默认0位置为有序，从1开始取) 。

  • 快速排序：通过中间值分成两个区域，左边区域比中间值小，右边区域比中间值大。

  • 希尔排序：按照步长(依次变小，最后一定是以1来排序)，左边的数与右边的数进行比较，小的数往右挪。

  • 归并排序：不断将列表分半分成一个个有序的个体，最后排序合并为一个新的列表。

搜索

  • 搜索是在一个数据集合中找到一个特定数据的算法过程。搜索通常的答案是真或假，找出该数据是否存在。
  • 二分查找:不断折半查找。

树

  • 树：根在上，一个节点含有的子树的个数称为该节点的度，一棵树中，最大的节点的度称为树的度
二叉树
  • 二叉树是每个节点最多有两个子树的树结构。
  • 二叉树的遍历：深度优先和广度优先。

  • 深度优先:一般用递归

  • 广度优先：从树的根节点开始，从上到下从从左到右遍历整个树的节点，一般用队列。
